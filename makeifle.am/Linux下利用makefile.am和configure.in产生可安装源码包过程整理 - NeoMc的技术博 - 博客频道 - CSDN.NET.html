Linux下利用makefile.am和configure.in产生可安装源码包过程整理

分类： Linux Intel 2010-12-09 18:50 873人阅读 评论(0) 收藏 举报

注：configure.in和configure.ac是同一文件的不同称呼，configure.in是比较早的版本。

先前条件：当前目录下有源码和已经写好的makefile.am

 

    1. autoscan

 

    生成了configure.scan，autoscan.log文件

 

    2.将configure.scan 修改为 configure.in

 

    然后修改configure.in中的内容。网上比较多的一个版本修改的内容如下：

    查了些资料，上面的AC_INIT和AC_OUTPUT是必须有的，用于定义configure命令执行时的开始接口和输出的文件。AC_PROG_C是编译c程序必须有的。还可以定义一些其他的宏用于将来检查编译环境。如果只是简单的工程不需要多加什么修改。

    3.aclocal

    生成 aclocal.m4 和 autom4te.cache (生成aclocal.m4的过程中涉及到configure.in)

   

    4.autoconf

    生成 configure (根据 configure.in, 和 aclocal.m4)

 

    5.编写makefile.am（因为假设源码中已经有了makefile.am所以略过）

 

    6.automake –add-missing

    这一步可能会提示找不到NEWS README AUTHORS ChangeLog文件，执行下面命令：

    touch NEWS README AUTHORS ChangeLog

    可能提示找不到config.in.h文件，执行下面命令：

    autoheader

    至此一个源码安装包已经全部build完毕，可以使用configure && make && make install来安装了

   

    7. ./configure

    这里如果后面加参数--prefix=/..//    可以指定安装路径

 

     8. make

    在符合GNU Makefiel惯例的Makefile中，包含了一些基本的预先定义的操作：

    make：根据Makefile编译源代码，连接，生成目标文件，可执行文件。

    make clean：清除上次的make命令所产生的object文件（后缀为“.o”的文件）及可执行文件。

    make install：将编译成功的可执行文件安装到系统目录中，一般为/usr/local/bin目录。

    make dist：产生发布软件包文件（即distribution package）。这个命令将会将可执行文件及相关文件打包成一个tar.gz压缩的文件用来作为发布软件的软件包。它会在当前目录下生成一个名字类似“PACKAGE-VERSION.tar.gz”的文件。PACKAGE和VERSION，是我   们在configure.in中定义的AM_INIT_AUTOMAKE(PACKAGE, VERSION)。

    make distcheck：生成发布软件包并对其进行测试检查，以确定发布包的正确性。这个操作将自动把压缩包文件解开，然后执行    configure    命令，并且执行make，来确认编译不出现错误，最后提示你软件包已经准备好，可以发布了。

    make distclean：类似make clean，但同时也将configure生成的文件全部删除掉，包括Makefile。

    9. make install

Flashcache does not properly reflect the physical sector size of the underlying device

No milestone
No one is assigned

When using flashcache on a 4k-sectored device, the resulting cached device falsely claims to be 512-byte sectored:

[root@nas72 ~]# flashcache_create -p back fc_data2 /dev/sda3 /dev/sdb -b 4k
cachedev fc_data2, ssd_devname /dev/sda3, disk_devname /dev/sdb cache mode WRITE_BACK
block_size 8, md_block_size 8, cache_size 0
Flashcache metadata will use 800MB of your 15762MB main memory
[root@nas72 ~]# blockdev --getpbsz /dev/sdb
4096
[root@nas72 ~]# blockdev --getpbsz /dev/mapper/fc_data2
512

I think this behavior could lead mkfs tools into wrong decissions, if not controlled manually.

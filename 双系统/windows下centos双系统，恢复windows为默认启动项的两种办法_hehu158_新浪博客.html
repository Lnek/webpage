windows下centos双系统，恢复windows为默认启动项的两种办法

(2014-02-25 11:57:57)
转载▼
分类： php
    笔者是在windows8.1上安装centos6.5，参考的是http://wenku.baidu.com/view/4711edeb195f312b3169a54f.html,最为重要的还是引导问题。
    在windows下安装centos成功后，如果不按键盘上的任何键它默认是直接进centos系统的，如果要切换windows原操作系统的话就比较麻烦了，开机就要开始按键盘了，一直按着某个键就可以切换了，选择other即可进入windows系统。由于硬盘的第一个扇区，即为MBR，主引导记录的修改，由grub进行引导系统，从而启动。现在的问题是我经常使用的是windows系统，所以我想默认启动windows系统。

方法一：不太推荐，保留了grub引导
    进入centOS下打开终端vi /boot/grub/grub.conf编辑配置文件如下：
default=0
timeout=5
splashimage=(hd0,6)/grub/splash.xpm.gz
hiddenmenu
title CentOS (2.6.32-431.el6.x86_64)
root (hd0,6)
kernel /vmlinuz-2.6.32-431.el6.x86_64 ro root=UUID=51422cb8-9980-4576-9899-fe8585546f36 rd_NO_LUKS  KEYBOARDTYPE=pc KEYTABLE=us rd_NO_MD crashkernel=128M LANG=zh_CN.UTF-8 rd_NO_LVM rd_NO_DM rhgb quiet
initrd /initramfs-2.6.32-431.el6.x86_64.img
title Other
rootnoverify (hd0,0)
chainloader +1

    正常在第一title后，会有第二个title，是windows的，在此可以把default=0改为default=1（1为第二个title，也就是windows设为默认启动）。
    如上设置重启后，会直接在一个other的选项，从而进入windows引导启动，默认还是grub在引导。
 
方法二：推荐，不保留grub引导
    如方法一修改完后基本已达到目的，可是经常会有人还会出现这样一个问题，假如linux哪天给删掉了，而开机的时候还是grub在引导，就会出现grub>的提示符，使你不得不手动引导等问题，并且习惯了看windows的启动项，还是比较喜欢那个，所以需要修改回windows引导程序来默认引导。这样一来，以后删掉linux也没有后顾之忧了（提示：删linux可以借助第三方工具删除,因怕windows不支持linux分区）。
    在此我们大家先顺便了解下windows启动的大致过程，首先加电bios程序启动自检→→将MBR代码读入内存→→找到bootmgr(xp系统中是找ntldr) →→BCD(然后从bcd中读取Boot Manager菜单的语言版本信息，再调用Bootmgr与相应语言的BOOTMGR.EXE.MUI组成相应语言的启动菜单，然后显示启动项) →→选择windows系统→→bootmgr去找系统分区，windows/system32/winload.exe加载操作系统内核。
    在此我将会用到GRUB4DOS这个软件，GRUB4DOS是一个优秀的双(多)系统引导软件，使用GRUB4DOS可以很方便的引导各种操作系统，比如DOS、Windows、 Linux等。
    现在的目的就是把它加入windows的启动项中，然后再修改GRUB4DOS相关（需要这个启动软件启动CentOS系统用）。接下来的操作都可以在windows系统下，唯一需要的就是centOS下的grub.conf的文件内容，如果没有拷贝，windows下还可以通过安装Linux_Reader查看linux的分区来查看。
    GRUB4DOS下载地址： http://download.gna.org/grub4dos/
    windows系统下下载并解压出文件后，需把grldr.mbr，grldr和menu.lst复制到C盘根目录下，然后以管理员身份运行cmd，打开命令提示符，利用bcdedit（启动配置数据储存）来添加相关项。
    输入bcdedit /create /d "CentOS6.5" /application bootsector在存储中新建项并应用到启动项中，如下图所示，会产生“标识符”。


{f4010382-9dbf-11e3-8291-94de80c7a1d6}就是我们介绍的标示符，然后直接输入如下的语句。
bcdedit  /set 标识符 device  partition=C:
bcdedit  /set  标识符 path  \grldr.mbr
bcdedit  /displayorder  标识符 /addlast
然后输入bcdedit.exe，就会看到多了刚刚编辑的一项
如果需要删除多余的项可以输入bcdedit /delete 后面跟查看到的标识符
 
然后打开编辑刚刚复制的menu.lst文件，把linux系统下的/boot/grub/grub.conf文件内容拷贝过来即可，如下图所示，是我menu.lst文件的内容，windows8.1下c盘根目录下是无法修改的，可能是权限问题，最好拷贝到桌面进行修改，然后在覆盖。


以上步骤都是为windows下能启动linux做的，此时我们还是grub引导，而非windows引导。这里还需要下载一个Mbrfix.exe(MBR修复软件)，然后以管理员身份运行cmd，打开命令提示符后，找到下载的mbrfix.exe程序（我放在C盘根目录下，cd..两次），最后输入MbrFix /drive 0 fixmbr /yes 对MBR进行修复，这样我们才算真正做到windows默认引导了。
 
然后重启你就发现非常成功了哈^_^

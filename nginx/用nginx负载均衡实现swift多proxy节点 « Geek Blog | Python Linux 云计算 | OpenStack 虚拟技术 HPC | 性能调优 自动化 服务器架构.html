用nginx负载均衡实现swift多proxy节点

2012-08-23日 | 标签: nginx | 作者：Geek

   Send article as PDF   

 

最近一直在研究swift，都是在PC机上进行的测试，性能不错，唯一就是网络依赖比较高。
关于swift的多proxy方案，想必都知道sina的架构。
一个负载均衡点，每个存储上跑着proxy服务。
nginx的负载均衡配置相当简单，网上的资料一大堆。

1.下载nginx源码包和nginx一个“411 Length Required”错误的包

wget http://www.longgeek.com/wp-content/uploads/2012/08/openstack/nginx-0.6.33.tar.gz
wget http://www.longgeek.com/wp-content/uploads/2012/08/openstack/nginx-0.6.33-length_required.patch

2.创建nginx用户

useradd -c "Nginx User" -s /sbin/nologin -r -d /var/lib/nginx nginx

3.安装

mv nginx* /usr/local/src/
cd /usr/local/src/
tar zxvf nginx-0.6.33.tar.gz
cd nginx-0.6.33
yum -y install *pcre* patch gcc make zlib* openssl*
patch ./src/http/ngx_http_request.c < ../nginx-0.6.33-length_required.patch

./configure
  --user=nginx
  --group=nginx
  --prefix=/usr/share
  --sbin-path=/usr/sbin/nginx
  --conf-path=/etc/nginx/nginx.conf
  --error-log-path=/var/log/nginx/error.log
  --http-log-path=/var/log/nginx/access.log
  --pid-path=/var/log/run/nginx.pid
  --lock-path=/var/log/lock/subsys/nginx
  --with-http_stub_status_module
  --without-poll_module
  --with-http_gzip_static_module
  --with-http_realip_module
  --with-http_ssl_module

make
make install

4.修改配置文件/etc/nginx/nginx.conf（修改红色部分）

cat /etc/nginx/nginx.conf | grep -v ^$ | grep -v .*#
worker_processes  1;
events {
    worker_connections  1024;
}
http {
    include       mime.types;
    default_type  application/octet-stream;
    sendfile        on;
    keepalive_timeout  65;
    client_max_body_size 1024M;       #限制用户上传大小为1G
    upstream swift {
       server 192.168.0.231:8080;
       server 192.168.0.232:8080;     #231、232和233都是proxy的ip
       server 192.168.0.233:8080;
    }
    server {
       listen 8080;
       server_name swift;
    location / {
       proxy_pass http://swift/;
    }
        error_page   500 502 503 504  /50x.html;
        location = /50x.html {
            root   html;
        }
    }
}

有个nginx.conf.defaulg会影响nginx.conf

mv /etc/nginx/nginx.conf.default /etc/nginx/default

4.运行nginx

/usr/sbin/nginx
netstat -ltunp | grep 8080

记得只需要在endpoint里指向nginx,其它proxy不需要指定。

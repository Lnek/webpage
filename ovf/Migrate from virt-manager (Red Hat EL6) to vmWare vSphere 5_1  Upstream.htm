Migrate from virt-manager (Red Hat EL6) to vmWare vSphere 5.1

Tom Van den Berg
January 10, 2013
No Comments

I’m using virt-manager on an RHEL6 environment and I’m going to migrate to a new vmWare vSphere environment. This migration isn’t straight forward.

# virt-manager

VmWare Convertor doesn’t support RHEL6 (Red Hat 6 or CentOS 6) and I’m not able to migrate with the convertor.

What I need to do is converting the disk of the virtual machine to a supported format, this can be done with the tool qemu-img.

If qemu-img isn’t installed, install it via yum.

# yum install qemu-img.x86_64

The virtual machine that I want to migrate is called Pandora, the disk is also called Pandora. To check the disk of the virtual machine, run following command.

# qemu-img info pandora 
image: pandora
file format: raw
virtual size: 40G (42945478656 bytes)
disk size: 0

As you can see, I’ve got a disk of 40G, so I need at least 40G of free space to do the conversion.

I’ve added a new disk to the system and mounted it to /var/migration

# df -h

/dev/mapper/vg_challenger-LVM_var
                      3.9G  233M  3.5G   7% /var
/dev/mapper/vg_challenger-solaris_iso
                      9.9G  2.4G  7.1G  25% /solaris_iso
/dev/mapper/vg_test-lvol0
                       99G  188M   94G   1% /var/migration

Now I can convert the disk to another format, for example vmdk.

# qemu-img convert <disk> -O vmdk <target_file>

In this case, it will be

# qemu-img convert pandora -O vmdk /var/migration/pandora.vmdk

When finished (this can take a while, depending on the size of your virtual machine), the new disk will be available in the folder /var/migration.

# ll -h

Now we need to move the .vmdk file to the vsphere server. I upload it via the vsphere client (via my local pc) by opening the client and browse to the datastore. Then I created a folder with the name of the virtual machine and put the vmdk file in it.

Go for a coffee (or 2) while uploading, and when it’s finished create a new virtual machine with the vSphere client and select the existing disk as boot disk.

Create a new virtual machine (Ctrl+N)

Select Typical …

Fill in the name of your new virtual machine and select next.

Select a host in your cluster and click next

Select a destination storage and click next (this is not that important because we’re going to remove this disk in one of the following steps, we’re not going to create it eventually).

Select the Guest Operationg System and click next

Select your network settings (for example, add an extra network device and change the network type) and click next.

Fill in the disk properties (not important because we’re not going to create it eventually) and click next.

Now we have to select “Edit the virtual machine settings before completion” and click on continue !

Click on add and select disk …

Select “Use an existing virtual disk” and click on next.

Browse to the uploaded vmdk file and select it.

Click on finish to add the new hardware to the virtual machine.
We have now two disks, we remove the first disk (select it and click remove) and keep the added disk that points to our vmdk.

The new virtual machine will be created.

Boot your virtual machine.

Because we migrated the Virtual Machine to new hardware, the hardware addresses are changed, not only the mac and uuid but also the device name (ex. ifcfg-eth0 -> ifcfg-eth3)

Change the content of the file (remove the old mac-address, change the device name) and rename the files to the correct device name.

You can find these files in /etc/sysconfig/network-scripts/.

Restart your network and you’re back in business.

Tip. install vmware-tools !

Converting into other hypervisor formats

General

Redbooth On-Premise is available using the industry standard OVF format supported by VirtualBox, VMware, Red Hat, IBM, Oracle and others. For easier deliverability it is packaged as an OVA package.

The .ova file is simply a tar package consisting of a .vmdk disk image, an .ovf manifest and an md5 checksum. As certain other hypervisors do not directly support the OVF/OVA format, in any conversion you do, you will need to convert the VMDK disk image file directly.

To extract the disk image you need to unpackage the ova:

tar xvf redbooth-onpremise-stable-5.1.5-blinking-planetary-entp.ova

Select one of the following hypervisor sections for conversion details:

KVM / QEMU

In order to conserve disk space and bandwidth, we use the Sparse form of the VMDK disk format. Older versions of QEMU do not support the VMDK Sparse format (Later releases of QEMU do have support for VMDK Sparse)

You can determine if your version of QEMU supports VMDK Sparse by executing the following command:

# qemu-img info redbooth-onpremise-disk1.vmdk
qemu-img: Could not open 'redbooth-onpremise-disk1.vmdk'

If the disk image details are displayed then no further conversion is necessary and you can use the disk image directly in KVM.

If you get a message like the above then VMDK Sparse is not supported and you should follow the steps below to convert the Redbooth On-Premise ova file into the qcow2 format supported by KVM:

1) Install VirtualBox. Free download from Oracle.com

2) Convert the redbooth onpremise vmdk disk into the VDI intermediate format:

VBoxManage clonehd redbooth-onpremise-disk1.vmdk --format VDI redbooth-onpremise-disk1.vdi

3) Convert the intermediate VDI disk into the QCOW2 format using qemu-img convert:

qemu-img convert -O qcow2 -c redbooth-onpremise-disk1.vdi redbooth-onpremise-disk1.qcow2

HyperV

HyperV uses a different disk file format than vSphere, VBox or KVM. A conversion to VHD is needed.

Follow these steps:

1) Install VirtualBox. Free download from Oracle.com

2) Convert the redbooth onpremise vmdk file into the VHD format

VBoxManage clonehd --format VHD redbooth-onpremise-disk1.vmdk redbooth-onpremise-disk1.vhd

Create a new virtual machine using the VHD file as a disk. Make sure the VM meet the requirements detailed here: Hardware requirements
